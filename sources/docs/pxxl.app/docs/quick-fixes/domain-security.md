If a Pxxl-purchased domain needs stricter security, use the domain **Settings** page.

## [Security Controls](https://pxxl.app/#security-controls)

You can:

- Force HTTPS.
- Force `www` traffic to the root domain when you want one canonical hostname.
- Enable or disable WebSocket upgrades.
- Add edge security headers.
- Turn on WAF checks for SQL injection, XSS, path traversal, and bad bots.
- Limit requests per IP or per domain.
- Allow or block IP ranges.
- Allow or block country codes.
- Restrict HTTP methods and content types.
- Add request or response headers.
- Enable maintenance mode during planned work.

## [Apply Changes](https://pxxl.app/#apply-changes)

After saving, click **Resync Proxy** so the active route receives the latest settings immediately.

For the full SSL and proxy settings reference, see [Domain proxy and SSL settings](https://pxxl.app/docs/quick-fixes/domain-proxy-ssl-settings).
