The Pxxl CDN API gives your backend direct access to object storage, asset management, and edge analytics. Use it to upload build artifacts, serve static files from edge cache nodes, track monthly bandwidth consumption, and pull proxy traffic logs for security analysis — all without leaving your deployment workflow.

This page covers the raw HTTP endpoints. If you would rather use a typed client, jump to the [Node SDK](/api/cdn-node-sdk) or [Go SDK](/api/cdn-go-sdk). For ready-to-deploy starter projects that already wire up uploads, see [Boilerplates](/api/cdn-boilerplates).

## Endpoints

| HTTP Method | API Path | Objective |
| --- | --- | --- |
| **GET** | `/api/v3/cdn/summary` | Query total bandwidth and remaining wallet credits. |
| **GET** | `/api/v3/cdn/assets` | Paginate through your uploaded asset catalog. |
| **POST** | `/api/v3/cdn/assets` | Upload a new asset to public or private spaces. |
| **GET** | `/api/v3/cdn/usage` | Read upload, download, delete, and credit events. |
| **GET** | `/api/v3/cdn/proxy-logs` | Retrieve traffic anomaly logs and edge events. |

## Upload an Asset

Send a `multipart/form-data` POST request to upload a file to your storage space. Set `visibility` to `public` for files served from edge cache nodes, or `private` for files accessed via pre-signed URLs.

```bash
curl -X POST https://gateway.pxxl.app/api/v3/cdn/assets \
  -H "Authorization: Bearer $PXXL_API_KEY" \
  -F "file=@./static/logo.png" \
  -F "visibility=public"
```

## Check Usage Summary

Query your current storage and bandwidth consumption, along with remaining wallet credits:

```bash
curl -H "Authorization: Bearer $PXXL_API_KEY" \
  https://gateway.pxxl.app/api/v3/cdn/summary
```

## Response Shapes

The raw HTTP API returns small envelopes such as `{ "success": true, "data": ... }`. The SDKs unwrap some of those envelopes for convenience.

| Operation | Raw HTTP response | SDK return value |
| --- | --- | --- |
| Summary | `{ success, data }` | `data` |
| List assets | `{ success, assets, pagination }` | Node/Python return the object; Go returns `assets`; Rust returns the raw object. |
| Upload asset | `{ success, asset }` | `asset` |
| Download asset | Binary response body | bytes/blob |
| Delete asset | `{ success, message }` | void/empty success |
| Usage | `{ success, events, creditLedger, credits }` | Node returns `events`; Python/Rust raw helpers return the object. |

## Asset Object

List, upload, summary `recentAssets`, and SDK upload calls return the same safe asset shape. Provider names, bucket names, raw R2 keys, and internal tenant paths are not exposed.

```json
{
  "id": "cm...",
  "userId": "cmb...",
  "projectId": "proj_123",
  "deploymentId": "dep_123",
  "storageName": "Pxxl CDN",
  "key": "cdn/file/cm.../logo.png",
  "fileName": "logo.png",
  "contentType": "image/png",
  "size": 18420,
  "publicUrl": "https://cdn.pxxl.app/cdn/...",
  "sha256": "8a5f...",
  "etag": "\"f1a2...\"",
  "visibility": "public",
  "kind": "file",
  "metadata": {
    "file-name": "logo.png",
    "kind": "file",
    "sha256": "8a5f...",
    "visibility": "public"
  },
  "createdAt": "2026-06-29T10:24:00Z",
  "updatedAt": "2026-06-29T10:24:00Z"
}
```

| Field | Type | Description |
| --- | --- | --- |
| `id` | string | CDN asset ID. |
| `userId` | string | Owner user ID. |
| `projectId` | string or null | Optional linked project. |
| `deploymentId` | string or null | Optional linked deployment. |
| `storageName` | string | Public storage label. |
| `key` | string | Safe display key, not the provider object key. |
| `fileName` | string | Sanitized original file name. |
| `contentType` | string | Uploaded content type or `application/octet-stream`. |
| `size` | integer | File size in bytes. |
| `publicUrl` | string | CDN URL. Public passive assets can use it directly. |
| `sha256` | string | SHA-256 digest calculated while staging the upload. |
| `etag` | string | Storage object ETag. |
| `visibility` | string | `public` or `private`. Unsafe active content is forced private. |
| `kind` | string | `file` or `artifact`. |
| `metadata` | object | Safe upload metadata. |
| `createdAt` | string | ISO timestamp for creation. |
| `updatedAt` | string | ISO timestamp for last update. |

## List Assets Response

`GET /api/v3/cdn/assets` accepts `page`, `limit`, `search`, and `kind`.

```bash
curl -H "Authorization: Bearer $PXXL_API_KEY" \
  "https://gateway.pxxl.app/api/v3/cdn/assets?page=1&limit=25&search=logo&kind=file"
```

```json
{
  "success": true,
  "assets": [
    {
      "id": "cm...",
      "storageName": "Pxxl CDN",
      "key": "cdn/file/cm.../logo.png",
      "fileName": "logo.png",
      "contentType": "image/png",
      "size": 18420,
      "publicUrl": "https://cdn.pxxl.app/cdn/...",
      "visibility": "public",
      "kind": "file",
      "createdAt": "2026-06-29T10:24:00Z",
```

`limit` is capped at `200`. If `page` or `limit` is invalid, Pxxl falls back to safe defaults.

## Upload Asset Response

`POST /api/v3/cdn/assets` stores the upload, records usage, charges cloud credits when enabled, and returns the created asset.

```json
{
  "success": true,
  "asset": {
    "id": "cm...",
    "storageName": "Pxxl CDN",
    "key": "cdn/file/cm.../logo.png",
    "fileName": "logo.png",
    "contentType": "image/png",
    "size": 18420,
    "publicUrl": "https://cdn.pxxl.app/cdn/...",
    "sha256": "8a5f...",
    "etag": "\"f1a2...\"",
    "visibility": "public",
    "kind": "file",
    "createdAt": "2026-06-29T10:24:00Z",
    "updatedAt": "2026-06-29T10:24:00Z"
  }

```

If the account lacks enough cloud credits, Pxxl returns `402`:

```json
{
  "message": "Not enough cloud credits for this CDN upload",
  "requiredCredits": 2,
  "balance": 0
}
```

## Download and Delete Responses

`GET /api/v3/cdn/assets/:id/download` returns the raw file bytes, not JSON. Pxxl sets safe headers such as `Content-Type`, `Content-Disposition`, `X-Content-Type-Options: nosniff`, and a locked-down content security policy.

`DELETE /api/v3/cdn/assets/:id` returns:

```json
{
  "success": true,
  "message": "CDN asset deleted"
}
```

## Get Proxy Logs

Retrieve edge traffic logs and security anomaly events recorded by the Pxxl CDN proxy:

```bash
curl -H "Authorization: Bearer $PXXL_API_KEY" \
  https://gateway.pxxl.app/api/v3/cdn/proxy-logs
```

Proxy logs include allowed and blocked request counts, risk index ratings, GeoIP concentration breakdowns, and the top IP sources contributing the highest traffic volumes to your project domains.

<Tip>
  If you accidentally expose a CDN upload API key in a client bundle, a public repository, or a build log, rotate it immediately from **Dashboard \> API Keys**. Delete the compromised key, create a replacement with the same CDN upload and read scopes, and update the secret in your backend environment before redeploying.
</Tip>
