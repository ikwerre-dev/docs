The CLI domain commands cover both operational checks and proxy analytics. Use them when you want to verify DNS, see whether a hostname is already used on Pxxl, connect a custom domain, or inspect traffic stats without opening the dashboard.

## List Domains

```bash
pxxl domains list
pxxl domains list --team <team-id>
```

Lists domains available to the authenticated account or selected spaceship.

## Check a Domain

```bash
pxxl domains check example.com
```

`pxxl domains check` combines local DNS checks with Pxxl-side ownership and proxy checks.

| Check | Source |
| --- | --- |
| DNS `A`, `AAAA`, `CNAME`, `NS` | Local DNS resolver. |
| HTTPS reachability | Public HTTPS request from your machine. |
| Already used | Pxxl domain and proxy route records. |
| Owned by you | Pxxl account or selected spaceship. |
| Proxy route | Pxxl global proxy binding state. |
| SSL | Pxxl route SSL state when available. |

The command returns statuses such as `available`, `owned`, `taken`, or `blocked`.

## Connect a Domain

```bash
pxxl domains connect
pxxl domains connect example.com --project <project-id>
pxxl domains connect app.example.com api.example.com --project <project-id>
```

Connect attaches the domain to the project and returns the expected DNS records needed for Pxxl routing:

```text
A     @     193.181.212.65
CNAME www   example.com
```

When you pass multiple domains, the CLI processes them one by one. Domains inside your plan allowance are connected, and domains above the allowance are shown as rejected with the plan limit and current usage.

After DNS propagates, verify the connection:

```bash
pxxl domains verify example.com --project <project-id>
pxxl domains activate <domain-id>
```

<Tip>
  For production domains, set DNS first, then keep checking with `pxxl domains check example.com` until the expected records resolve.
</Tip>

## Managed DNS Records

For Pxxl-managed domains, the CLI can list and edit DNS records.

```bash
pxxl domains records <domain-id>
pxxl domains records add <domain-id> --type A --name @ --value 193.181.212.65 --ttl 60
pxxl domains records update <domain-id> --record-id <record-id> --type A --name @ --value 193.181.212.66
pxxl domains records delete <domain-id> --record-id <record-id>
```

Pxxl validates record conflicts before saving. For example, the same hostname cannot be both a `CNAME` and an `A` record.

## Nameservers and Zone Status

```bash
pxxl domains nameservers verify <domain-id>
pxxl domains nameservers set <domain-id> --values ns1.example.com,ns2.example.com
pxxl domains nameservers reset <domain-id>
pxxl domains zone <domain-id>
pxxl domains switch-to-pxxl-dns <domain-id>
```

Use these commands when a domain is managed by Pxxl DNS or when you need to move a managed domain back to Pxxl nameservers.

## Certificates

```bash
pxxl domains cert <domain-id> --out example.com.pem
```

This downloads the public proxy certificate as a PEM file.

## Domain Stats

```bash
pxxl domains stats example.com
pxxl domains stats example.com --timeframe 30d
```

Domain stats show request count, bandwidth, latency, blocked requests, errors, top countries, and top cities using the same proxy analytics pipeline as the dashboard.

Supported timeframes:

```text
24h
48h
72h
7d
30d
```

## Permissions

Domain commands require an API key with `scope=domain`, `scope=domains`, `scope=reselling`, or `scope=all`.

| Operation | Required permission |
| --- | --- |
| `list`, `check`, `stats` | `read` |
| `connect`, `verify`, `activate`, DNS writes, nameserver writes | `read_write` |

## Plan Limits

If your plan only allows a fixed number of custom domains, `pxxl domains connect` will connect the domains that fit and show a clear rejection for the rest:

```text
domain        status  message                                             limit  used
extra.com     403     Your current plan allows up to 1 custom domains.     1      1
```

Upgrade the plan or remove an unused custom domain before adding more.
