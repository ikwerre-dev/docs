The public Pxxl repository includes deploy-ready boilerplates under `boilerplates`.

| Boilerplate | Use case |
| --- | --- |
| `static-cdn-gallery` | A fast static gallery that renders CDN-hosted assets. |
| Express | API starter with npm, pnpm, Bun, or Yarn variants. |
| Vite React | React app starter with npm, pnpm, Bun, or Yarn variants. |
| `static-cdn-gallery` | A fast static HTML/CSS gallery that renders CDN-hosted assets. |
| `php-basic` | Basic PHP app starter. |
| `hono-bun` | A tiny Hono API starter running on Bun. |
| `fastify-pnpm` | A Fastify API starter using pnpm. |
| `nextjs-pnpm` | A minimal Next.js App Router starter using pnpm. |
| `astro-npm` | A fast Astro content site starter using npm. |
| `python-fastapi` | FastAPI Python API starter. |
| `go-http` | Go standard-library HTTP service starter. |

Each boilerplate includes a `pxxl.boilerplate.json` manifest and an SEO-focused README that explains how to deploy the project on Pxxl.

```bash
pxxl init --new
pxxl init --new express-bun --name my-api --domain pxxl.pro
pxxl deploy
```

The CLI reads the manifest and writes matching `pxxl.toml` defaults for package manager, framework, port, install command, build command, and start command.
